A： 阿里实习面试笔记

从2月28号将邮件发给内推的师兄，到4月2号收到实习意向书，过了漫长的一个月，终于走完了阿里实习应聘的流程。这里记录一下中间的过程，以便自己总结，也希望能帮助到后来的人。

简历准备

简历的设计与编写

大二下学期的时候曾经抱着试一试的心态去微信面过实习生（不过面完对方告诉我公司规定不收大二，建议我明年再来），当时已经用 Adobe Illustrator 设计过一份简历（选择 AI 是因为排版方便，有很多好用的辅助功能，也很容易导出 PDF ），现在也觉得当时的设计还OK，所以就在原来的基础上改。

在大三上学期还没结束的时候就在宿舍弄好了简历的 PDF 版。按照以前在网上看到过的相关经验将长度控制在一页以内，头部放上姓名、联系方式和应聘职位，下面写上学校，英语水平，项目经验和技能。比较感慨的是这一年里简历上可以写的东西多了很多，去年简历确实是想填满一页都勉强，也没有什么实习经历，今年已经有两份实习经历可以写上去，技能方面也有了很多积累。不过自己的短板是没有做过什么项目，因为觉得做外包需求乱变很麻烦，追钱也很棘手，加上分散精力会影响学业（我自己定下的目标是大一大二都要拿到奖学金），所以就没做过什么外包。毕竟大学只有四年，离开了学校就很难有这么大段的时间沉下心来系统地学习基础知识了，在基础不牢的情况下去做上层的应用也是事倍功半，感觉有点得不偿失。

写简历的时候在应聘职位写的是“前端工程师/软件工程师”，没有丢掉后面那个是因为觉得自己在身为一个前端程序员以前，首先是个程序员。像 FLAG 这类公司一般都没有细分出所谓前端工程师的职位，他们要的都是 full stack 的工程师，title 一般也都是 software developement engineer 之类，所以自己简历上的 JD 感觉也不要只写前端工程师比较好。

学习方面自己的亮点是拿过两次奖学金（一次三等一次一等），虽然做技术不一定看重这个，但是能拿奖学金也是好学的表现之一，毕竟简历就是要尽量将自己适合职位的一面展示出来，所以也写了上去。GPA 需要去找教务处核实，我们学校的绩点算法比较特殊需要换算，而且大三课还没上完，所以就没写，不过应该是 Top 5% 以内，加上学校是 985，还是比较有竞争力的。

因为平时坚持看原版教材/技术书和在技术社区用英语交流，自己的四六级分数比较高（四级679，六级616），也是一个亮点，所以也在简历上突出了一下，用进度条的设计可以一目了然地看出得分率。作为补充，写上了能流畅阅读英语教材论文和文档，能无字幕看开发者大会和MOOC视频、在技术社区交流之类的说明，毕竟技术岗位看英语成绩也就是想看你能不能做这些事，干脆自己明着写出来。

工作经历方面只有在创业公司和IT咨询类公司分别做过两三个月的实习，就简单写了一下时间地点，列了一下负责的工作，高亮了一下用到了什么技术和特别的地方。之前也说过自己没什么项目，个人凭兴趣做的一些东西感觉也没有很了不起，干脆就没写。

虽然在知乎上看到过计子说技能方面最好简写，不过毕竟还在校，工作经历确实薄，所以这次的简历还是以技能描述为主。自己的策略是尽量将技能有条理地、系统地展现出来（也可以从侧面反映出你知道它们谁是谁，用来做什么，不至于表现得知识体系一团乱，基本概念都没搞懂），同时尽量表现出自己技能的广度（嗯毕竟深度还不够……）。基本上分为前端、后端和其他技能。其实写完之后发现自己基础最扎实的还是其他技能（Linux操作，C/C++ 和 Python 写各种各样的东西之类的），反而前端的经验还不足，写上去也不是很有底气。中间思考再三删掉了很多自己不够熟的东西（比如Haskell，后来听说有认识的人作死写了上去遇到懂行的悲剧了哈哈哈），毕竟万一真的问到了不了解的地方答不上来就不好了。

在设计方面，吸取上次在微信面试的教训，改用白色背景和灰度打印也不影响阅读的双色主题（因为现场面试需要打印简历出来）。原本因为能写的东西不多，尽量用 infographics 的方式展现内容，现在信息量大了不少，所以转回以文字描述为主，对技术名词和特殊的地方用亮色强调。除了文字以外还用了大量的图标辅助阅读，这里的图标使用了开源的 Font Aweson，因为 AI 可以在字体对应的字符集里选取字形，用起来很方便。正文主要使用了开源的思源黑体（其实用的是 Noto Sans），标题和一些点缀用了自己有个人授权的方正雅宋系列（方正淘宝官方店个人授权都是2块5块的，简直业界良心……）以及Font Squirrel 的一些 free font，总之就是尽量不使用没有授权的盗版字体。由于信息量的调整，从原来 1:1 的双栏布局改成了 5:7 左右的布局，借助 AI 的网格辅助线也很容易实现。

在线版简历

寒假期间在家完成了简历的在线版，毕竟应聘前端，在线简历也是体现个人技术能力的一种手段。在制作前期本来是打算做全屏动画的，但是因为一开始就打算做一个从 JSON 读取数据，自动生成页面的简历生成器，并且要能够自动适配双语，如果做成全屏动画版的话，很难在数据量有很大变动的时候动态调整布局又保持良好的视觉效果，所以最后还是打算将 PDF 版的设计直接搬到网页上，毕竟这个设计的响应式布局也很好做。

在线简历使用了 Jade 生成模板，Less 生成样式，用 Gulp 完成构建并且做成能够一行命令部署到 GitHub pages 的设计。在编写模板的时候尽量做到语义化，也就是尽量用 HTML5 语义标签，能不用 div 就不用 div，并且尽可能减少不必要的标签数。做出来的东西基本上标签以 dl 和 ul 为主（比如技能列表就多为 dl），囊括在多个 section 里，比较符合语义。用一些 Screen Reader 检查了一下，基本满意。

由于 HTML 方面尽量精简，所以 CSS 方面会做比较多的工作（比如时间轴的设计上每个标记的图标都是使用 CSS 伪元素完成样式的，并没有在 HTML 上为它添加多余的元素）。 Less 代码的编写大量参考了 Bootstrap 的源代码，做了个自动生成的网格系统，每个模块都分开文件，padding、margin、width和各种响应式断点尽量不硬写数字，而是统一作为变量放到一个 variable.less 里。为了方便扩展，选择器基本全是 class，类名的命名方式也受 Bootstrap 影响，有类似于命名空间的设计。受 BEM 风格的影响，尽量做到各个模块可以随意组合放置，不互相依赖。在整体设计上尽量提高可扩展性，方便以后添加新的模块，理想的情况是自己以后需要更新简历时，如果是在现有模块里添加新信息（比如新的实习、新的技能之类）可以完全不用动其他地方，直接更新一下 JSON 的数据即可。即使是添加新模块，也可以重用现有的代码，只不过多做一些指定新的名词对应什么图标这样比较 trivial 且必须人脑来做的工作。

面试过程

乌龙的内推冲突

大概 2 月底的时候技术社团里在阿里 ICBU（应该属于 B2B 的一部分）工作的师兄就发了邮件让大家要内推的把简历发给他。其实在此之前自己已经跟一个比较熟的天猫的师兄联系过，拜托他关注一下内推什么时候开始，不过看 ICBU 的师兄的说明，貌似部门内推不会互相冲突，于是就把自己的简历发过去了。事后才发现是会冲突的……=___,=

经过大概是这样的，2 月 28 号把简历交给了 ICBU 的师兄，3 月 3 号天猫的师兄说他们那边也开始了就把简历交给了他，3 月 4 号收到了 B2B 的内推邮件，上面写着

我可以接受多人推荐吗？ 不可以。如果你收到多个链接，请与推荐人确认后再提交简历，之后其他链接均会失效。

将信将疑地跑去问了一下天猫的师兄，我们猜不同部门应该不影响，然后我就点了邮件里的链接把之前在阿里招聘系统填过的简历挂钩。3 月 5 号天猫的师兄来通知我去收下内推的邮件，发现没有收到，然后他去问了一下 HR 发现悲剧了，要前一个 BU 的流程走完才能走下一个流程……所谓走完，不是挂就是过，不管是哪种好像再去天猫都比较困难了。其实基于天猫团队在知乎和微博上发的东西还有师兄的一些介绍，自己是对天猫的前端比较感兴趣的（移动/web 大前端的概念 + 正在使用 Node 做和淘宝 midway 类似的东西），不过既然已经进了流程了，那么既来之则安之，先拿到 offer 再说……

不过这次乌龙也不怪师兄们，他们也不知道 BU 内推之间会冲突，只能说，毕竟大公司，被流程坑很正常……不过阿里系那么多 BU，内推都只能投一个，感觉好浪费啊……之后在漫长的等待面试结果的那段时间里感觉要备胎了，忍不住去 V2EX 发了个求实习贴，结果大概有七八个阿里其他部门的人跑过来问有没有兴趣面试，都只能告诉他们已经走了这边的流程会冲突，虽然看他们介绍自己的团队感觉很动心也没办法……= =b

一面（技术面）

3 月 5 号下午 3 点左右接到了阿里总部打来的电话，对方直接问现在可不可以直接电面，当时我正准备去上课，吓了一跳……然后告诉他等下要上课，改约晚上 7 点。现在一想为了上课改约貌似显得有点怠慢，不过对方也不是很介意的样子。这是我这个实习招聘季接到的第一个面试通知，之前没想到这么快就会开始面试，所以没怎么准备，到了课室之后自然完全没心上课了，刚好带着电脑就开始做h5bp 的那个前端面试题集锦，不过课室没有网络所以也只挑了部分题来看看，总之就是坐立难安心脏乱跳地在课室里度过了两个多小时……下课之后也没时间吃晚饭，就跑回宿舍准备，不过当时很紧张也不知道该干啥，在电脑上瞎鼓捣了一下，到 7 点过了几分钟就接到了面试的电话。

一面的面试官一上来先让我做下自我介绍，我基本就把之前发内推的时候邮件正文写的个人介绍复述了一遍（就是学校、专业、工作方向和在哪里实习过什么岗位之类的）。估计面试官一上来也不太清楚问啥，于是他就问我 npm 怎么安装一个包……我内心囧了一下，不过上一份实习快走的时候自己帮忙面试接班人也是一开始不知道问啥好就问了些很囧的问题，估计他也跟我当时一样。我就简单回答了一下怎么安装本地和全局的包，还有自己常用的一些参数之类。接下来问了一下之前的两份实习都做了什么，有什么收获，然后他问我我的简历（PDF版）如果要做到网页上怎么实现，我立刻很兴奋地说其实已经做过了，然后把链接的位置告诉他（这时候我才发现我把在线版简历的链接放得太偏僻了，估计他第一眼没看到……）。

面试官打开在线简历的链接之后，就开始针对这个东西各种细致地问了……我估计了一下一面大概 50 分钟，起码有 30 分钟我们在聊跟这个相关的内容……从模板怎么划分模块和设计继承关系，到样式上怎么抽取同类的组件，到 CSS3 和 web font 的兼容性怎么处理，再到针对移动端怎么做响应式设计，还有怎么构建和部署，以及为什么要选择这样的技术栈，Jade/Handlebars等模板引擎的比较，Less 和 Sass 的比较之类，细细碎碎谈了一大堆东西。接下来再针对我简历上写的东西，聊了一下前端性能优化的技巧和前端模块化（比如 Sea.js）的机制。聊了大概 50 分钟之后就到了那个经典的问题——你有什么要问我的吗？（后来我发现阿里所有的面试官，包括 HR，都必问这一句=__=||），我就问到那边实习大概会负责什么样的工作。对方就介绍了一下他们团队做的事情，然后说相对于其他前端团队他们历史包袱比较轻，用新技术比较不受限制。我们聊着聊着就聊到了前后端分离，刚好那几天研究过淘宝、Linkedin 和 ebay 在使用 Node.js 做前后端分离上的实践，于是结合自己上一份实习的感受（那里也是 Java 做的后端，在前后端合作上跟前面那几家遇到过类似的困境）又侃了一会儿，最后就是惯例的“如果有消息的话我们还会通知你”，然后就挂电话了。

一面的面试官感觉非常有礼貌，我们开口的时点冲突的时候他都会停下来让我先说，给人印象很棒。

二面（技术面）

3 月 9 号早上大概 9 点多钟，走路去上课的时候看手机，发现一两分钟前手机有个未接来电，那段时间不知道为啥手机老是莫名其妙自动挂掉电话……一看上面显示的是阿里总部的号码就吓 cry 了，然后去打听了一下貌似如果没打通他们还会打回来，就提心吊胆等回电。

等到中午快 1 点的时候打了回来，因为在吃饭所以又请他等一会儿再打（现在一想貌似有点作死……），不过接下来也没心吃了，随便扒了几口就回到了宿舍。过了一会儿电话就响起来了。

因为之前打电话的时候他问了我岗位是不是前端程序员，我还以为这是HR……然后再接电话的时候我就很傻逼地问了他他是不是HR =___=||| 他说这个你先不要在意，总之我们就面一下。现在想来应该对方是部门的技术主管一类的人，不是做前端这块的所以才问了岗位。

二面问得比较泛，包括问我怎么接触前端编程的，怎么看待女生编程不多的问题（……），看了哪些书，为什么想来大公司，具体的问题貌似只问了一个 JS 跨域（临时忘光了OTZ，就回答了一个以前看人讨论过的 QQ 邮箱和微信用的 iframe 设 domain 的方法，事后才想起来我以前在服务器和客户端都写过 CORS 啊 = =）

到了经典问题的时候我继续用我一贯的回答（到那边实习大概会负责什么样的工作，后来我无论什么面试我都问这个666），对方回答主要会做数据可视化相关的东西，包括外部和内部的。

三面（技术面+交叉面）

大概 3 月 10 号左右就收到了三面面试官的短信来事先约时间，约到了 3 月 12 号晚上。快到约定的时间的时候对方又发了条短信道歉说在开会要推迟一下，又过了快半个多钟就打过来了（相比之下，差不多同时间段面的的某大厂的面试官各种不吭声放我鸽子，囧……）。

三面也是技术面，一上来让我自我介绍的时候让我介绍一下自己印象最深的项目和遇到的最大的挑战，不过回答的时候貌似把两个实习都说了一遍（异步控制，大量遗留CSS代码的重构和模块化）……可能是因为那段时间家里发生了一堆狗血天天心烦意乱，虽然这个面试官态度很好，可是我回答的时候老是答着答着发现自己不知道说到哪了囧，不过他貌似也不是很在意，都会顺着我的话头继续问。接下来主要是针对我的回答去问项目的细节和一些实现方面的难点（比如让我解释一下项目里怎么用 Promise 的，说一下 Promise 的原理，我很作死地又说了 Promise 就是个 Monad 的梗……），然后也是一些比较泛的问题，包括对团队的期望，想做什么样的工作，平时从哪里接触新技术等等。

回答经典问题的时候对方跟我介绍了一下这个是交叉面，类似实验的对照组，面试官不是你要去的团队的人，所以他也没办法回答我具体会干嘛，不过他还是大概介绍了一下可能会去做啥。

四面（HR面）

3 月 17 HR 面，实习经历，学习过程，职业规划，家庭对工作地点的看法，喜欢听还是说，兴趣爱好。

四面后才发现系统可以看进度

Offer

3 月 20 日下午接到阿里总部的电话，在上课没敢接，跟同学交流目测是确认实习时间的。等到 3 月 24 还没有再打，就发了封邮件给内推的师兄，下午就打回来确认了实习时间。

3 月 31 日下午收到了录用通知短信，4 月 2 日收到 offer 说明和实习意向书。

阿里的实习 Offer 待遇还是挺不错的，结合自己后来拿的 Offer，跟同学和其他认识的人交流了一下，发现国内的互联网大厂本科实习大概是 阿里 ≈ 网易 > 百度 > 腾讯。阿里高主要是因为待遇本硕不分，加上房补的话（比如去网易杭州）可能网易比阿里还高一些（毕竟土豪）。百度本身的工资和前面两个一个水平，但是在北京或者上海的话租房更贵，而且要求实习 6 个月却只给前 3 个月的房补。腾讯其实你可以跟他谈，我的面试官问了阿里给的待遇之后表示可以补到一个水平（吓cry……实习生也能谈价钱？！），虽然因为家里的狗血还是没有选择留在广东。

另外阿里的实习似乎是几家大厂里时间要求最松的……拿了 offer 不来也没关系，秋招也可以直接绿色通道进终试，也没有特别要强留你转正的意思，实习时间也没有太大的要求。百度和网易都希望实习至少 6 个月，腾讯不确定能来一般不发 offer 而且希望毕业能留下来。阿里只有一个指定时间前实习断续 60 工作日 / 连续 45 工作日才能直接转正否则要走秋招的要求，来不来，时间多长，转不转正，这些都交给你自己选择。
